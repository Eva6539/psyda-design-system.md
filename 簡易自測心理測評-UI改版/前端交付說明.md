## 2. 技術與字體

- 純 HTML / CSS / 原生 JS，無框架依賴；互動用少量 vanilla JS（`IntersectionObserver`、`classList`、`addEventListener`）。
- 字體（Google Fonts）：
  ```html
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Fraunces:opsz,wght@9..144,400..600&family=Plus+Jakarta+Sans:wght@400..700&display=swap" rel="stylesheet">
  ```
  標題 = Fraunces（襯線）、內文 = Plus Jakarta Sans；中文自動 fallback 到 PingFang TC / Noto Sans TC。
  **正式版建議導入一套中文品牌字**（思源宋體/黑體等），質感更統一。

## 4. 互動與動效規格

| 互動 | 行為 | 實作要點 |
|------|------|---------|
| **打卡喚醒 Mochi** | 預設貓睡在窩裡（冒 z）；按「開始心晴打卡」→ 貓醒來走出窩、開始走動 | 切換 class：隱藏睡貓 → 顯示走動貓並加 `.awake`（觸發 roam/leg/tail 動畫） |
| **貓走動** | 醒來後在場景內來回走、走到邊緣轉身、偶爾眨眼 | CSS `@keyframes roam`（translateX）+ `face`（scaleX 翻面）+ 四腳/尾巴/眨眼動畫 |
| **點貓 / 摸摸餵食陪玩** | 冒愛心、貓跳一下、顯示提示、陪玩升親密度 | JS 動態 `createElement` 愛心（**勿用 innerHTML**），加一次性 `.jump` |
| **聲波錄音** | 麥克風脈動光環 + 起伏聲波條 + 倒數 | 多根 `<i>` 各設 `height` 與 `animation-delay`，`scaleY` 跳動 |
| **充電生成中** | Mochi 睡覺 + 進度條，文案「約 1 分鐘內完成」 | indeterminate 進度條動畫；實際完成由後端通知 |
| **完成頁** | 開心 Mochi 跳跳 + 閃亮 + 飄愛心 | bounce + sparkle + floatUp |
| **心理能量愛心** | 報告 hero：愛心填充比例＝% | 目前為 AI 圖示意；正式版建議用**可控向量愛心**做動態填充（見第 6 點） |
| **指標長條** | 進場/捲動可見時由 0 填到分數 | `IntersectionObserver` → 對長條 `i` 加 `.is-filled`（`width: var(--w)`） |
| **進場/捲動揭示** | 卡片依序淡入上移 | `.ds-reveal` + `IntersectionObserver` 加 `.is-in`，`transition-delay` 做 stagger |

> 已加 `prefers-reduced-motion` 降載。

## 5. 重要結構提醒

- **底部導覽列要放在「捲動容器之外」**，用 `position:absolute; bottom:0` 釘在手機底部；放進捲動區會跟著捲到中間（原型已修正）。
- 捲動條隱藏：`scrollbar-width:none` + `::-webkit-scrollbar{display:none}`。
- 顆粒質感：容器疊一層 SVG `feTurbulence` noise、低透明度 multiply（見原型 `.screen::after`）。

## 6. 資料與邏輯（依業主 PDF）

**測評**：題數 **1 題**、語音作答、生成時間目標 **1 分鐘內**（後端）。

**五項情緒指標**（各 1–10 分，分數越高代表該狀態越明顯/風險越高）：
`心情低落度` `情緒活躍度` `擔憂緊張度` `情緒敏感度` `身心疲憊度`

**心理能量（百分比 + 等級）**
- 五項加總範圍 5–50。
- **百分比公式（demo 暫用，待業主確認）**：`電量% = (50 − 五項總分) / 50 × 100`
- PDF 原始 5 級（電池語意，**命名待業主確認**，demo 已柔化）：
  `滿格充電 0–10` / `穩定續航 11–20` / `電量不足 21–30` / `低電量警示 31–40` / `即將關機 41–50`（依總分落點）
- 前端呈現：愛心填充高度 = 電量%；旁顯示百分比與等級文字。

**調節建議（AI 生成，後端 prompt；50 字內）**
> 請根據五項指標分數，生成 50 字內調節建議。優先針對分數最高的 1–2 項，給溫和、具體、可執行的建議。語氣支持、非診斷、不恐嚇（避免「你有病」「嚴重異常」）。**若任一指標 ≥ 8 分，加入「建議尋求專業支持」。**

**專屬療愈內容（AI 生成，後端 prompt；50 字內）**
> 第二人稱（你）、像溫柔的陪伴者；強調理解、接納、關懷與希望；不分析分數、不提數字、不用診斷詞彙；以情感支持為主。直接輸出療癒文字。

> 前端只負責呈現後端回傳的「建議文字」「療愈文字」「五項分數」「電量%與等級」；上面 prompt 規格給後端參考。

## 7. RWD / 手機友善

- 原型固定 360×780 是為了 demo；正式版請改成**滿版 RWD**（`100vw / 100dvh`、安全區 `env(safe-area-inset-*)`）。
- 觸控目標 ≥ 44px；長者友善建議提供加大字級模式。
- 字級、對比度依無障礙標準檢查（暖底上的淺色文字要留意對比）。

## 8. Mochi 素材

| 素材 | 用途 | 形式 |
|------|------|------|
| `mochi-half-v3.png` | 報告 · 抱半滿愛心 | AI 插畫 PNG（1920×1088）|
| `mochi-ai-warm.png` | 抱實心愛心（流程說明/頭像來源）| AI 插畫 PNG |
| `mochi-sleep.png` | 睡覺（可用於空狀態）| AI 插畫 PNG |
| 走動貓 / 蜷睡貓 / 慶祝貓 | 首頁與流程的「會動卡通貓」 | **內嵌 SVG**（在 HTML 內，可改色/動畫）|

**Mochi 兩種形態（刻意分工）**：
- 互動畫面（首頁、流程）= **會動的卡通 SVG 貓**（可程式控制動畫/狀態）。
- 最終報告 = **AI 精緻插畫**（靜態高質感）。

> 目前 AI 圖為**示意素材**。建議正式版請插畫師依此風格出一套統一的 Mochi（含各姿勢與去背），前端只需替換圖檔；報告的愛心也建議改成**可控向量**以做動態填充。
