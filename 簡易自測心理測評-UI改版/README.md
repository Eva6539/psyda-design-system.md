## 一、本次範圍（這次做了什麼）

針對「**簡易自測心理測評**」功能，做整套**手機端 UI 改版**，導入療癒寵物 **Mochi（橘貓）** 的陪伴世界觀。涵蓋三大畫面 + 設計系統 + 前端交付規格 + 素材 + 給業主的設計提案簡報：

| 區塊 | 內容 |
|------|------|
| **首頁** | 寵物陪伴區：Mochi 預設睡在貓窩，使用者「心情打卡」後醒來走出窩、可摸摸/餵食/陪玩；本週心情、今日金句、心情打卡、探索服務、固定底部導覽 |
| **測評流程** | 單頁多畫面切換：說明 → 語音作答（聲波動畫）→ 充電生成中 → 完成（慶祝動畫）。題數依業主需求＝ **1 題** |
| **報告** | 心理能量（Mochi 抱「半滿愛心」示意 50%）、五項情緒指標、調節建議（任一指標 ≥8 顯示「建議尋求專業支持」）、專屬療愈（Mochi 想對你說） |
| **設計系統** | 共用設計 token 與元件樣式（`交付-mochi-design-system.css`）|
| **前端交付規格** | `前端交付說明.md`：檔案對照、字體、token、互動規格、資料邏輯、RWD、待業主確認事項、後端清單 |
| **設計提案** | 給業主的完整簡報（PPTX / PDF / HTML 三種形式）|

> 與既有 PSYDA skill 的差別：那份是「PSYDA 題庫管理系統」的設計規範 skill；**本資料夾是「簡易自測心理測評」這個功能的 UI 改版設計稿與交付包**，兩者是不同的東西。

## 三、怎麼看

- **業主 / 看設計**：開 `簡易自測心理測評-設計提案.pdf`（或 `.pptx`）。
- **前端工程師**：先讀 `前端交付說明.md`，三個 `.html` 原型為純 HTML/CSS/原生 JS，可直接當實作參考。
- **看可互動原型**：建議用本機伺服器開啟（原型引用了相對路徑圖檔與 Google Fonts），例如在此資料夾執行：
  ```bash
  python3 -m http.server 8000
  # 瀏覽器開 http://localhost:8000/首頁-寵物-融入精緻版.html
  ```
  直接用 `file://` 開亦可顯示，但以本機伺服器體驗最完整。

## 四、技術重點

- 純 **HTML / CSS / 原生 JS**，無框架依賴。
- 字體：Fraunces（標題襯線）＋ Plus Jakarta Sans（內文），中文 fallback PingFang TC / Noto Sans TC。
- 配色：暖色療癒「Forest Canopy」色系（細節見 CSS 與交付說明）。
- Mochi 兩種形態刻意分工：**互動畫面用會動的卡通 SVG 貓**；**報告用 AI 精緻插畫**。

## 五、待業主確認（會影響實作，詳見交付說明第 9 點）

1. 心理能量 5 級命名（PDF 原電池語 vs demo 柔化版）。
2. 心理能量百分比公式（暫採 `(50 − 五項總分) / 50 × 100`）。
3. 是否採用 Mochi 養成世界觀（PDF 之外的加值設計）。
