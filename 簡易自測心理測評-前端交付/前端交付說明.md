## 0. 一分鐘總覽

- **識別色**:暖珊瑚 `#F5806C`(主按鈕、home FAB、重點)。粉(`#F6EBEE` 柔霧粉)是品牌中性底,**不是**識別色。
- **材質語言**:**黏土(立體物件:icon/表情球/按鈕)× 液態玻璃(卡片表面)**。
- **字體**:Noto Sans SC / PingFang(乾淨無襯線);文字錨點 `#4A3340` 深莓灰,不用純黑。
- **token**:見 [`tokens.css`](tokens.css),`<link>` 引入即可用 `var(--xxx)` 與 `.ds-*` helper。

## 2. 設計 Token（摘要,完整見 `tokens.css`）

```
識別色   --brand-coral #F5806C / light #FFB597 / deep #E8654F
中性底   --surface-mist #F6EBEE   表面玻璃 --surface-glass
輔助色   --mint #5CBB8C  --lavender #A98FE0  --butter #FFC945  --mochi #F3A557  --rose #E58FAC
語意色   --risk-low #7FC6A0  --risk-mid #F3BD6A  --risk-high #F08A72
文字/線  --ink #4A3340  --ink-2 #8A7382  --ink-3 #BBA9B6  --line #ECE0E8
圓角     卡片22 / 按鈕16 / 磚&icon14 / 膠囊999
陰影     暖粉色調、向下、可分兩層(環境光+主投影);避免四周均勻發胖
間距     8pt;頁邊16–18、卡內距16–18、區塊間距24
```

**材質規則**
- **玻璃**(卡片表面):半透明白 + `backdrop-filter:saturate(165%) blur(17px)` + 頂部亮邊高光 + 暖色柔投影。不支援 backdrop-filter 會自動退化成半透明白卡。
- **黏土**(立體物件):球面 radial 高光 + 內外柔陰影,做出膨膨立體;用於 icon、表情球、按鈕、吉祥物、心理電量。
- **背景**:柔霧粉 + 2 道很淡的暖/冷光暈(餵玻璃折射)+ 極淡紙質顆粒(`feTurbulence`,降 AI 感)。

## 3. 互動與動畫規格

| 項目 | 行為 | 實作要點 |
|------|------|---------|
| **心情球體 · 跳動** | 選中當天心情後,該黏土球**輕柔上下浮跳** | `@keyframes`:`translateY(-3px) ↔ -11px`,1.5s ease-in-out;**浮跳非硬彈跳**;見 `.ds-sphere.is-on` |
| **本周心情** | 每日打卡。已過+今日才有天氣(小玻璃方塊),**未來日留空**(faint 小點);今日加**珊瑚圈**;指示器放固定高度槽 → 七天標籤對齊 | 玻璃小方塊用黏土高光;今日 `border:2px solid var(--brand-coral)` |
| **答題 · 語音** | 三段狀態(機器人念題 → 輪到你 → 錄音中) | 即時聲波(多根 `<i>` scaleY + delay)、倒數環(conic-gradient mask)、「我說完了」可提早結束、可重新作答、題目常駐、可重念題目 |
| **生成中 · 敘事進度** | 分步驟切換(聆聽你的聲音 → 整理五項情緒 → 為你寫下建議)縮短「體感時間」 | 步驟狀態 done/active/todo;`setInterval` 推進(示意);實際完成由後端通知 |
| **完成** | 短慶祝 + 寵物回饋(心理電量+、親密度+) | 彩屑 + 黏土貓 + chip |
| **心理電量 · 水波填充** | 報告愛心填充面是**晃動水波**(雙層、不同速度) | 正弦波 path + `translateX` 平移循環,clip 在愛心內;填充高度 = 電量% |
| **指標長條** | 進場由 0 填到分數 | `IntersectionObserver` 加 class 觸發 `width:var(--w)` |
| **進場/捲動** | 卡片淡入上移、依序揭示 | reveal class + IntersectionObserver + transition-delay |

> 全部動效都加 `@media (prefers-reduced-motion:reduce)` 降載(`tokens.css` 已示範)。

## 4. 資料與邏輯（依業主需求 PDF）

**測評**:題數 **1 題**、**語音作答**、每題 30 秒、生成目標 **1 分鐘內**(後端)。

**五項情緒指標**(各 1–10 分,分數越高代表該狀態越明顯/風險越高):
`心情低落度`、`情緒活躍度`、`擔憂緊張度`、`情緒敏感度`、`身心疲憊度`

**心理電量(百分比 + 等級)**
- 五項加總範圍 5–50。
- **百分比公式(demo 採用,待業主拍板)**:`電量% = (50 − 五項總分) / 50 × 100`
- PDF 五級(電池語,**命名待確認**,demo 已示範柔化版):
  `滿格充電 0–10` / `穩定續航 11–20` / `電量不足 21–30` / `低電量警示 31–40` / `即將關機 41–50`(依總分落點)
- 前端:心理電量愛心填充高度 = 電量%;旁顯示百分比與等級。

**調節建議(AI,後端 prompt;≤50 字)**
> 依五項指標,優先針對最高 1–2 項給溫和、具體、可執行建議;支持、非診斷、不恐嚇。**任一指標 ≥ 8 分,加入「建議尋求專業支持」**,並做成**可點 → 導向 app 內專業諮詢**。

**專屬療愈內容(AI,後端 prompt;≤50 字)**
> 第二人稱、溫柔陪伴;強調理解/接納/關懷/希望;不分析分數、不提數字、不用診斷詞彙。

> 前端只負責呈現後端回傳的「五項分數、電量%與等級、調節建議、療愈文字」;上述 prompt 規格給後端參考。

## 5. 報告呈現原則（對象 = 通用人士,含企業/學生/長者）

- **層次/漸進揭露**:由上往下(心理電量 → 五項 → 建議 → 療愈 → 下一步)。
- **五項加質性標籤**(平穩/需留意/偏高)+ 圖例,並非只給冷小數;顏色語意一致(綠/琥珀/紅)。
- **命名柔化**:電池語偏負面、與「不恐嚇」衝突 → demo 用「需要充充電」等(待業主拍板)。
- **無障礙**:提供「放大字級」與「語音播報報告」(對長者/閱讀困難者友善)。
- **下一步 CTA**:預約專業諮詢 / 記錄今天 / 回首頁,不要 dead-end。

---

## 6. RWD / 無障礙

- 原型用固定手機畫布(規範頁為文件寬版);正式版改**滿版 RWD**(`100dvh`、安全區 `env(safe-area-inset-*)`)。
- 觸控目標 ≥ 44px;主操作沉到**底部拇指區**。
- 暖底上的淺色文字注意對比;支援加大字級。

## 7. 待業主拍板（會影響實作）

1. 心理電量 5 級命名:電池語(滿格充電…即將關機)vs 柔化版(需要充充電…)。
2. 百分比公式:PDF 兩處範例不一致,demo 採 `(50−總分)/50×100`,請拍板。
3. 是否採用寵物(Mochi)世界觀與「測評結果回灌寵物狀態」的加值設計。
